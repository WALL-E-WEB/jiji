# 技术调研过程总结

## 调研过程

1. 明确需求

   ```
   获取需求场景，有利于理解需求；
   ```

2. 实现需求方案分类

   ```
   优先关注于缺点，再关注于优点。
   缺点知否致命，优点是否必要。
   ```

3. 各分类的具体实现流程

   ```
   资料收集整理；优先参考官方文档。
   ```

4. 实现的差异
